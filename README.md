# Lean Six Sigma Procurement Process Improvement

This repository presents a complete Lean Six Sigma process improvement project focused on improving the procurement purchase request to purchase order process. The project follows a structured six-week improvement journey covering process mapping, root cause analysis, project planning, Lean tool implementation, performance measurement, and sustainability planning.

## Project Overview

The selected process is the Procurement Purchase Request to Purchase Order Process. The main problem addressed in this project is delayed purchase order approval caused by incomplete purchase requests, manual email-based approvals, late budget checks, limited process visibility, and lack of KPI monitoring.

The project applies Lean Six Sigma principles to identify process waste, analyze root causes, design improvements, measure performance, and create a sustainability plan for long-term process control.

## Project Objective

The objective of this project is to reduce purchase order approval delays, improve request accuracy, increase approval visibility, reduce manual follow-up, and support continuous improvement in procurement operations.

## Process Problem

In the current-state scenario, purchase requests that should be approved within 2 to 3 business days may take 7 to 10 business days. The delays create operational inefficiencies, supplier coordination issues, employee frustration, and additional administrative workload for procurement and finance teams.

## Key Performance Indicators

The project measures improvement using the following KPIs:

* Purchase order approval cycle time
* Incomplete purchase request rate
* Approval delay rate
* Manual follow-up emails per request
* On-time purchase order approval rate
* Process visibility and tracker completion rate

## Expected Improvements

The proposed improvements are expected to achieve the following outcomes:

* Reduce purchase order approval cycle time from 7 to 10 days to approximately 3 to 4 business days
* Reduce incomplete purchase requests through standardized digital forms
* Improve approval visibility using a Kanban-style tracker
* Reduce manual follow-up emails through clearer ownership and approval SLAs
* Improve procurement reliability through KPI monitoring and continuous review
* Sustain improvements through SOP updates, feedback loops, and regular process audits

## Tools and Techniques Applied

### Process Mapping

A current-state process map was created to understand the flow from purchase request submission to purchase order approval.

### Root Cause Analysis

The 5 Whys and Cause and Effect Diagram were used to identify the main causes of delays, including lack of standardization, manual approvals, late budget checks, and weak KPI tracking.

### DMAIC Project Planning

The project follows the DMAIC structure:

* Define the problem, scope, and objectives
* Measure current process performance
* Analyze root causes
* Improve the process using Lean tools
* Control the improved process using KPIs and sustainability measures

### Lean Tool Implementation

Value Stream Mapping, 5S, and Kanban were selected to reduce waiting time, improve document organization, and create better process visibility.

### Performance Measurement

A simulated dataset was used to compare pre- and post-implementation performance. Trend analysis, variance analysis, and control chart concepts were applied to evaluate improvement.

### Sustainability Planning

A continuous improvement framework was developed using PDCA, KPI reviews, employee feedback, refresher training, SOP updates, and process ownership.
